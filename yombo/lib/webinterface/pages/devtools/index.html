Developer Tools

Developer tools allows you to create new types of automation items. This includes new commands, device types, and modules that are available. Settings here do no directly affect the local gateway.

  • Commands - Commands are what are sent to the automation devices to control them.
  • Device Types - Each device type can accept certain commands (on, off, dim). Most device types require settings such as device address, these are are called 'device variables'.
  • Input Types - Input types specify what information a command needs in order to function. For example, a 'set brightness' command would need to how how birght to set the lamp. Input types are mapped inside the device types.
  • Modules - Modules extend the capabilites of the system. The modules section manages what device types the module can interact with. Like device types, modules can have 'module variables' which allows end users to configure the module. Configuration examples include what USB interface to talk with, an IP address to reach the device typs it controls, or what email server to use.

Developer tools allow you to manage various items that control how the automation system system works. For more, click the help button above.

Local gateway settings are in a different area.

  • Commands - Commands are sent to devices to control them.
    • Index - List commands available.
    • Add - Add a new command.
  • Device Types - Devices (such as lamps, appliances) are classified into device types.
    • Index - List device types available.
    • Add - Add a new device type.
  • Input Types - How user input is collected and processed.
    • Index - List input types available.
    • Add - Add a new input type.
  • Modules - Manage local and global modules.
    • Index - List modules available.
    • Add - Add a new module.
